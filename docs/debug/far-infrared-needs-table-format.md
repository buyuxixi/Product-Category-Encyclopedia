# 远红外 3.2 表格格式异常

## 问题描述

远红外热疗「03 用户需求与品类痛点 → 3.2 消费者购买决策路径」表格中，部分加粗文案显示为原始 Markdown 星号，例如：

- `*专业评测站**`
- `*$100-$250为主流甜点价格带**`
- `*EMF测试视频**`

## 排查过程

1. 前端 `renderMarkdown` / `inlineFormat` 对合法 `**text**` 可正常产出 `<strong>`
2. 查本地库 `encyclopedia_sections`（`FAR_INFRARED` / `needs`）发现正文加粗标记已损坏
3. 对比源文件 `data/far_infrared.md`：源文件为正确的 `🆕 **专业评测站**` 等写法

## 根因

库内 `needs` 内容与源 Markdown 不一致：`🆕 **...**` 在入库/编辑过程中被截断为 ` *...*` / ` *...**`，导致加粗正则无法匹配，星号原样显示。

## 修复方案

1. 以 `data/far_infrared.md` 第三节为权威内容，回写本地 `encyclopedia_sections.content`
2. 顺带清理第三节列表项 `-  **` 多余空格与行尾空白

## 验证结果

- 库内已无奇数个 `**` 行
- 关键短语已恢复为 `🆕 **专业评测站**` 等合法加粗
- `inlineFormat` 抽样渲染产出 `<strong>`，无残留 `**`
